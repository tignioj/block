反向代理子路径时JS以及CSS丢失解决方案

发表于 2020-02-29 | 分类于 nginx

1. 需求：

客户访问tignioj.top
服务器将请求反方向代理到二级路径
github.tignioj.io/blog

2. 正确的配置

1. 关键

！注意路径的斜杠，不能写错
proxy_pass https://tignioj.github.io/blog/;
和
proxy_pass https://tignioj.github.io/blog;
两个是不一样的
前者代表把代理路径的根路径作为访问路径的根路径
后者代表仅仅访问该接口！

2. 解释

第一段：

客户访问 http://tignioj.top:80;的时候，ngxin帮我们访问了proxy_pass指向的路径

于是当我们访问
https://tignioj.top

nginx返回了如下的页面
https://tignioj.github.io/blog/;

第二段

location ^~/blog/ {
   proxy_pass https://tignioj.github.io/blog/;
}

注意，这里的/blog/ 对应了代理服务器的二级路径

因为原网址 https://tignioj.github.io/blog/ 的所有css，js路径都是相对于/blog/目录下的
所以当我们打开
https://tignioj.top

主页会把https://tignioj.top当作根路径，然后去访问css, js等资源
比如
https://tignioj.top/blog/js/index.js

由于我们第一段配置了
https://tignioj.top/指向 https://tignioj.github.io/blog/;

这样就会使得js的请求路径实际上变成了
https://tignioj.github.io/blog/blog/js/index.js

因此，我们需要告诉nginx，当有人请求/blog/这个二级目录下面的页面时，重新分配路径

参数解释

  • ^~/blog/ ：匹配所有 blog开头的url 即 所有https://tignioj.top/blog/开头的url
  • proxy_pass https://tignioj.github.io/blog/; : 当请求匹配的url时，将请求转到 https://tignioj.github.io/blog/
